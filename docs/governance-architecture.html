Capability Extraction

When a run initiates, the orchestrator retrieves the work item and resolves its bounding capability boundary.

  • Database column: parentCapabilityId on work_items
  • Refreshes from local cache: capabilities_cache
  • Governing check: Governing capabilities are excluded from work-item routing.

Prisma Schema Declarations

Local sync of Singularity IAM capabilities, storing name, type, and governing flags:

model Capability { id String @id // IAM capability_id name String type String? status String? isGoverning Boolean @default(false) syncedAt DateTime @default(now()) @@map("capabilities_cache") }

Point-in-time snapshot mapping constraints, ensuring policies don't retroactively rewrite execution audits:

model GovernanceOverlaySnapshot { id String @id @default(uuid()) workItemId String? workflowInstanceId String? workflowNodeId String? governedCapabilityId String overlayHash String resolvedOverlayJson Json resolvedAt DateTime @default(now()) @@unique([workItemId, workflowNodeId, overlayHash]) }

Approved waivers that exempt the agent loop from specific blocking constraints in the policy:

model GovernanceWaiver { id String @id @default(uuid()) workItemId String? workflowInstanceId String? workflowNodeId String? controlKey String // e.g. "ALLOW_UNVERIFIED_EDIT" reason String status String @default("REQUESTED") // APPROVED | REJECTED }

Policy Enforcement at Tool Gateway

Enforcement runs as a hard coded double-validation logic inside context-fabric:

  1. Wire Descriptor Check: When resolving turn tools, turn.py only advertises the union of tools allowed by the active policy, preventing standard LLMs from planning unauthorized actions.
  2. Server-Side Gate: If an agent attempts to execute a tool outside its current phase scope (e.g. trying to run apply_patch during EXPLORE), the tool call is intercepted by tool_gateway.py and rejected with a POLICY_BLOCKED code.
  3. Validation Verification: Before completing stages, the system evaluates the edit provenance and verifies file target modifications against the plan via path_coverage.py.
