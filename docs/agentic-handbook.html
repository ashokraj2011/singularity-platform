❖ 1. High-Level Architecture

Singularity enforces a strict execution segregation rule: Business engines orchestrate; Sandboxes execute. Context-Fabric sits at the center of the lifecycle, owning state persistence and policy checking while delegates execute dumb tool scripts.

🧠

Stateful Context-Fabric

Written in Python. Manages multi-turn orchestration (`stage_driver.py`), single-turn execution (`turn.py`), state transition checks (`phase_state.py`), and validation oracles (`loop.py`). Owns all context and state caches.

📦

Stateless MCP Server

Written in TypeScript. A lightweight, sandboxed tool host exposed via `/mcp/tool-run`. Serves files and runs isolated terminal commands. Possesses zero understanding of the active stage, phase state, or history.

🎭

Prompt Composer

Serves as the prompt assembly library. Keeps reusable templates, lessons, stage prompts, and prompt profiles. Compressed by a sidecar stopwords compressor when limits are exceeded.

🛠 4. Interactive Deep Dive: M74 Core Restorations

Milestone 74 restored and enhanced key validation gates that were dropped during the M71 transition. Click through the tabs below to explore the mechanics and Python implementations of these gates.

Chronological Monotonic Numbering

To prevent LLM context-window exhaustion, history is compressed. Verbatim turns are capped at 8, while older turns collapse to `[TURN-N-RECAP]` breadcrumbs. Because breadcrumbs utilize a user role, they get classified under the `prelude` slice during subsequent calls. M74 introduced a lookup to scan existing breadcrumbs and offset indices, avoiding turn-indexing resets.

# history_compression.py
def _count_breadcrumbs(prelude: list[dict[str, Any]]) -> int:
    count = 0
    for msg in prelude:
        if msg.get("role") != "user":
            continue
        content = msg.get("content") or ""
        if content.startswith("[TURN-") and "-RECAP]" in content:
            count += 1
    return count

EditReceipt Provenance Binding

To resolve the loophole where an agent could claims changes in `EditReceipt` without actually executing edits, `loop.py` scans successful mutating dispatches (`apply_patch`, `replace_text`, etc.), pulls their `code_change_id` envelopes, and validates that claimed edits are backed by real tool executions.

# loop.py
def _check_edit_receipt_provenance(*, receipt: dict, produced_changes: dict) -> list[str]:
    edits = receipt.get("edits") or []
    unbacked = []
    for entry in edits:
        path = entry.get("file")
        canonical = _normalise_change_path(path)
        if canonical not in produced_changes or not produced_changes[canonical]:
            unbacked.append(path)
    return unbacked

Synthesizing Verifiers & Preserving Failure Logs

After edits are completed, the orchestrator automatically runs test discovery (`recommended_verification`) and triggers unit test execution (`run_test`) on behalf of the agent. Test output summaries are fed into the prompt history. To prevent truncation from discarding actual failures, M74 preserves the log **tail** rather than the head.

# verify_synthesis.py
def _summarise(text: Any, *, max_len: int = 1500) -> str:
    s = str(text)
    if len(s) <= max_len:
        return s
    # Truncate setup logs, keep the error traces and tracebacks
    return "...[truncated earlier logs]\n" + s[-max_len:]

Self-Correction Validation Retry Budget

Validation failures (such as missing a required field in `PlanReceipt`) were historically treated as fatal, aborting the run with `VALIDATION_BLOCKED` on turn 1. `stage_driver.py` now implements a validation retry budget, allowing the agent one opportunity to receive structured error details and self-correct.

# stage_driver.py
if turn.step.validation_error and not turn.step.phase_advanced:
    consecutive_validation_errors += 1
    if consecutive_validation_errors > _VALIDATION_RETRY_BUDGET:
        result.stop_reason = "VALIDATION_BLOCKED"
        return result
    # Feed error feedback directly into next turn prompt
    history.append(_render_validation_error_message(turn.step.validation_error))
    continue

Windows Normalization & Verification Gate Enforcement

Key changes addressing platform loopholes and OS-specific behavior:

Component Gaps Addressed Implementation Detail
Windows Paths Backslashes (`\`) broke path comparisons on Windows, resulting in false-positive `PHASE_COVERAGE_GAP` blocks. `_normalise` and `_normalise_change_path` now proactively replace `\` with `/` prior to prefix removal.
Verification status loophole Agents could bypass checks by submitting a `"passed"` status with failed test commands inside `commands_run`. `VerificationResultPayload` now executes a model validator asserting every command in `commands_run` has `exit_code == 0` when status is `"passed"`.
PLAN $\rightarrow$ SELF_REVIEW edge Non-coding workflow nodes (e.g. Architect planning, Product Owner intake) were blocked from finishing. Modified transition map to support bypassing EXPLORE/ACT/VERIFY stages if the stage policy limits are set to planning-only.

🔒 5. Multi-Turn PII Masking Engine (`pii_mask.py`)

When the platform migrated to stateless tool dispatches, the session-level PII mapping was lost. `context-fabric` now owns Option A (stateful orchestration-side masking), preserving security boundaries across turns and resume events.

Mask/Unmask Flow Stateful Mapping

The mapping maintains strict correspondence between values and token indexes (e.g. `[EMAIL_1]` always corresponds to `a@b.com` throughout the stage run). This allows the agent to reason about identity across turns without ever exposing unregulated PII to external LLM providers.

Args Unmasking

Walks arbitrary JSON arguments recursively, executing regex replacements on string leaves longest-token-first, so `[EMAIL_10]` does not get corrupted by substring matches against `[EMAIL_1]`.

def unmask_string(text: str, token_map: dict) -> str:
    tokens = sorted(token_map.keys(), key=len, reverse=True)
    pattern = re.compile("|".join(re.escape(t) for t in tokens))
    return pattern.sub(lambda m: token_map.get(m.group(0)), text)

Result Masking

Performs regex detection over dotted IPv4 quads, ZIP+9, NANP phone numbers, credit card digits (Luhn-validated), emails, and SSNs, generating sequential token identifiers and updating the mapping.

def _luhn_valid(s: str) -> bool:
    digits = re.sub(r"\D", "", s)
    total = 0
    alternate = False
    for ch in reversed(digits):
        n = int(ch)
        if alternate:
            n *= 2
            if n > 9: n -= 9
        total += n
        alternate = not alternate
    return total % 10 == 0
